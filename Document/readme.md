# 📂 Project Documentation

This folder contains the project documents prepared during different phases of the **UCAB – MERN Stack Cab Booking System** development lifecycle.

| **Document** | **Description** |
|--------------|-----------------|
| 🧠 **Brainstorming & Ideation** | Records the initial project ideas, problem statement, objectives, proposed solution, and feature discussions. |
| 📋 **Requirement Analysis.docx** | Describes the functional requirements, non-functional requirements, system requirements, use cases, and project scope. |
| 📅 **Project Planning.docx** | Contains the product backlog, sprint planning, user stories, story points, sprint schedule, project tracker, velocity, and burndown chart. |
| 🏗️ **Project Design.pdf** | Includes the system architecture, UML diagrams, ER diagram, database design, workflow diagrams, and UI design. |
| 📖 **cab_booking.docx** | Final project report covering implementation, testing, results, conclusion, and future enhancements. |

---

## 🎯 Purpose

The documents in this folder provide a complete record of the planning, analysis, design, and implementation phases of the UCAB project. They serve as supporting documentation for understanding the project's development process and overall system architecture.

> **Project:** UCAB – MERN Stack Cab Booking System
